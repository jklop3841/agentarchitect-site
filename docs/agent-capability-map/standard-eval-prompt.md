# Agent Capability Map Standard Evaluation Prompt

请基于真实任务表现，而不是印象、宣传文案或空泛赞美，完成这份 Agent 能力测评。

要求：
- 必须基于至少 2-3 个真实任务样本填写。
- 不允许只写优点，必须写清不适合任务、边界与风险。
- 不允许跳过失败模式、稳定性或成本感知。
- 如果某项信息无法确认，请明确写出“不确定”与原因，不要猜。
- 输出必须克制、具体、可比较，避免“非常强”“什么都能做”这类空话。

请围绕以下维度输出：

## 基本身份
- Agent 名称
- 角色定位
- 主要工作方式
- 典型输入 / 输出

## 可调用工具 / 环境
- 它能直接调用哪些工具、接口或运行环境
- 它依赖哪些外部条件才能稳定工作
- 哪些能力只是理论上可做，哪些是你已经验证过的

## 擅长任务
- 它最擅长完成的任务类型
- 哪些任务里它的表现明显优于通用助手
- 最适合把它放在哪个工作流节点

## 不适合任务
- 哪些任务交给它风险很高
- 哪些任务会浪费成本、时间或沟通带宽
- 哪些任务看起来相关，但其实不应该交给它

## 边界与风险
- 它最常见的失败方式是什么
- 它对上下文、工具、权限、格式的依赖边界是什么
- 哪些情况下必须加人工复核

## 输出稳定性
- 同类任务重复执行时是否稳定
- 是否容易出现风格漂移、漏项、误解约束或执行断裂
- 你对它输出一致性的总体判断是什么

## 速度 / 成本感知
- 它更适合快而轻的任务，还是慢而重的任务
- 在你观察里，它的 token / 工具 /协作成本感知如何
- 什么情况下它是“值得调用”的，什么情况下不值得

## 推荐使用方式
- 最推荐的调用姿势是什么
- 给它任务时应该提供什么前置信息
- 怎样写 brief 或约束能明显提高表现

## 一行结论
- 用一句话总结：这个 Agent 最适合在什么边界内，被怎样使用。
