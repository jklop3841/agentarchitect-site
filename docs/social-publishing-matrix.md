# 自媒体同步发布可行性矩阵

来源文件：`C:\Users\97566\Desktop\自媒体平台账号矩阵管理表.xlsx`

目标：同一套内容生成管线，每天生成中文文章，控制在 `900-1500` 字，并带 `GEO` / 智能体架构相关标签；网站发布后，同步分发到账号矩阵里的平台。

## 总体结论

可以做，但不能理解成“一个脚本无差别发 15 个平台”。

正确架构是：

1. 内容生成层：每天抓 `AI 资讯速览`、`AI 论文简报` 做选题雷达，再独立查官方/论文/项目源。
2. 主站发布层：写入网站 `src/lib/content.ts`，保留完整文章、来源和标签。
3. 分发适配层：把同一篇文章转成不同平台需要的标题、正文、标签、封面和格式。
4. 平台发布层：每个平台一个 adapter，用 Playwright / Cookie / 持久浏览器会话发布。
5. 审计层：记录发布状态、失败原因、URL、是否需要人工补发。

## 现有工具状态

已有本地技能：

- `media-auto-publisher`
  - 支持：百家号、搜狐号、知乎、微信公众号、小红书、抖音号。
  - 当前能力更偏：平台信息、Cookie 管理、导航流程生成。
  - 还需要补：通用标题/正文/标签/发布按钮填充逻辑。

- `toutiao-publisher`
  - 支持：头条号。
  - 已有高级发布脚本，支持标题、内容文件、封面、headless。
  - 当前问题：本机虚拟环境缺 `patchright`，需要先修依赖。

当前不具备：

- 全平台已登录 Cookie。
- 全平台字段选择器。
- 全平台发布成功后的 URL 回收。
- 风控/验证码/扫码登录自动处理能力。

## 统一发布管线建议

### 文章生成标准

- 中文。
- 每篇 `900-1500` 字。
- 必须带 `GEO` 相关标签。
- 每篇至少包含：
  - 标题。
  - 2-3 句摘要。
  - 正文。
  - 标签。
  - 来源链接。
  - 平台短标题版本。
  - 平台摘要版本。

### 内容变体

同一篇文章分三种形态：

1. 主站版：完整文章，保留来源和结构。
2. 技术社区版：900-1500 字，强调工程、Agent 架构、MCP、GitHub、RAG。
3. 泛资讯平台版：900-1200 字，降低技术密度，保留判断和结论。

### 标签建议

基础标签：

- `GEO`
- `智能体架构`
- `AI工作流`
- `AI资讯`
- `AI论文`

按内容追加：

- `MCP`
- `代码代理`
- `GitHub`
- `RAG`
- `Agent评测`
- `工具协议`
- `企业AI`

## 风险和边界

- 平台可能有验证码、扫码、风控、编辑器改版，不能承诺永远无人值守。
- 每个平台都应该保存发布日志，失败不能阻塞主站发布。
- 不建议一开始就全平台自动点击“发布”，可以先做“自动填充草稿 + 人工确认”试运行。
- 通过 3-7 天稳定试运行后，再逐步打开自动发布。
- 平台账号安全优先于自动化速度。
