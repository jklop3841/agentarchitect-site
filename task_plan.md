# 多平台同步发布自动化计划

目标：把网站每日 AI 文章生产管线扩展为多平台中文同步分发系统。文章控制在 `900-1500` 字，必须带 `GEO` 标签，优先服务智能体架构、AI 工作流、GEO、GitHub 证据层和技术社区分发。

## Phase 1：固化矩阵与发布策略

- [x] 读取桌面账号矩阵 Excel。
- [x] 将平台分为 A/B/C 三个自动化优先级。
- [x] 生成可读实施矩阵：`docs/social-publishing-matrix.md`。
- [x] 生成机器可读平台配置：`docs/social-publishing-accounts.json`。
- [x] 记录现有工具能力和限制。

## Phase 2：修复第一批发布运行环境

- [ ] 修复 `toutiao-publisher` 依赖，确认 `patchright` 可用。
- [ ] 跑通头条号登录状态检测。
- [ ] 确认通用 `media-auto-publisher` 能保存/读取 Cookie。
- [ ] 为微信公众号、百家号、知乎、搜狐号建立登录态检查步骤。

## Phase 3：内容分发稿生成

- [ ] 建立平台分发稿结构：标题、摘要、正文、标签、来源、封面。
- [ ] 将每日主站文章压缩成 `900-1500` 字中文分发版。
- [ ] 强制每篇包含 `GEO` 标签和智能体架构相关标签。
- [ ] 支持技术社区版和泛资讯平台版两种文本密度。

## Phase 4：第一批平台草稿模式

- [ ] 头条号：自动填入标题、正文、标签，先不自动点击发布。
- [ ] 百家号：自动填入标题、正文、标签，先不自动点击发布。
- [ ] 知乎：自动填入标题、正文、标签，先不自动点击发布。
- [ ] 搜狐号：自动填入标题、正文、标签，先不自动点击发布。
- [ ] 微信公众号：自动填入标题、正文、封面/摘要，先不自动群发。
- [ ] 记录每个平台的选择器、弹窗、登录失效和失败原因。

## Phase 5：发布日志与失败恢复

- [ ] 新建发布日志目录。
- [ ] 每个平台记录：文章 slug、标题、平台、状态、时间、URL、错误信息。
- [ ] 平台失败不阻塞主站发布。
- [ ] 每日自动化 inbox 汇报同步发布状态。

## Phase 6：第二批技术社区扩展

- [ ] CSDN adapter。
- [ ] 掘金 adapter。
- [ ] 腾讯开发者社区 adapter。
- [ ] 阿里云开发者社区 adapter。
- [ ] 博客园 adapter。

## Phase 7：自动发布开关

- [ ] 每个平台独立配置 `draft` / `publish` 模式。
- [ ] 通过 3-7 天草稿模式稳定验证后，再逐步打开自动发布。
- [ ] 保留高风险平台的人工确认模式。

## 当前默认策略

- 第一阶段不全平台自动点击发布。
- 先做“主站自动发布 + 多平台自动生成/填充草稿”。
- 等账号登录态、选择器和平台风控稳定后，再打开部分平台自动发布。
