# 智能体项目报价前确认表

作者：智能体架构师卢成 / Jack Lu  
主站：https://agentarchitect.me

## 1. 客户原始需求

- 客户说他想要什么？
- 他希望解决什么业务问题？
- 是否有明确验收标准？
- 是否有预算范围？

## 2. 交付物确认

请明确写出要交付什么：

- 页面/系统/工作流：
- 文档/培训/模板：
- 是否包含部署：
- 是否包含后期维护：
- 是否包含员工培训：

## 3. 修改范围

- 免费修改次数：
- 每次修改的定义：
- 超出修改如何收费：
- 需求变更如何处理：

## 4. 售后范围

- 售后周期：
- 售后包含什么：
- 售后不包含什么：
- 是否提供长期顾问：

## 5. 客户责任

- 客户需要提供哪些资料？
- 客户需要提供哪些账号/权限？
- 客户需要安排谁配合？
- 客户延迟提供材料如何处理？

## 6. 报价判断

- 基础报价：
- 风险加价：
- 加急加价：
- 长期维护费用：
- 是否建议先做诊断：

## 7. 是否应该接单？

- 是否边界清楚：
- 是否付款明确：
- 是否客户靠谱：
- 是否存在灰色风险：
- 是否值得接：
